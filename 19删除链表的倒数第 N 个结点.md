# 思路
既然是倒数第n个，我们不妨从正数第n个开始想也就是设置一个计数器，遍历，然后过一个结点count++，然后把第n个节点删除。这样如果头尾相反，那么我们就有了结果。即我们要先获取尾结点，然后往前删即可。
一个容易混淆的点，即正数的话我们是从i=1 开始，也就是实际删除的位置。但如果是倒数删第n个，即length-n+1 才是我们要删除的位置，因为length-n 比方说4-1=3 ，这样只是把从后往前n个也是1个元素删掉，i=3，并不是倒数第1个，这时候如果使用数组下标思想也就是从0开始记录它的删除位置刚好  
如果想让index=实际删除的位置也就是4-n+1，我们就要从1开始去数而不是从0开始，因为既然我们要删除的结点是第n个。那这个就可能是0-n 也就是实际要删除的结点的位置是从0-n，如果正数index对应遍历的cur也就是1=第一个元素，2=第2个元素。一切正常，0也进入不了循环，设计哑结点也能完成。
如果是倒数第n个，我们很自然想到length-n，但是length-n实际上所代表的位置实际上从后面开始要删除的位置的前一个。因此如果是1对1的绑定 也就是 1对应cur1 2 对应cur2，那么遍历的结束条件是index = length-n + 1
```
public ListNode deleteReverseNum(ListNode head, int deleteReverseIndex){

        //获取链表长度
        ListNode cur = head;
        int count = 0;
        while (cur != null){
            cur = cur.next;
            count++;
        }


        //获取道长度后再次遍历，正着遍历找到倒数第n个结点也就是正着数第count-n个结点，同时记录它的前驱，用于删掉当前节点并
        //使得前驱接上后继结点。
        cur = head;
        int index = 0;
        //设置哑节点防止删除头结点，头结点没有前驱。
        ListNode dummy = new ListNode(-1,head);
        ListNode prev = dummy;
        while (cur != null && index < count - deleteReverseIndex){
            cur = cur.next;
            prev = prev.next;
            index++;
        }
        prev.next = cur.next;
        return dummy.next;
    }
```

```
        ListNode dummy = new ListNode(-1,head);

            index++;
            if(index < count - deleteReverseIndex + 1){

            }else {

            }
```
